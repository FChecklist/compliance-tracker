# VERIDIAN AI — Mobile App Template

`veridian-mobile-template.html` is the canonical reference UI for every VERIDIAN AI product branch (GRC, Sales, HR, SCM, Project Management, and any future vertical). It is a static, self-contained mock — open it directly in a browser, no build step. It exists so that every new product built on the VERIDIAN AI platform starts from the same interaction language instead of each team re-inventing navigation, ergonomics, and AI-conversation scope from scratch.

This is the 14th and final iteration of a live design process (see `veridian_landing_v2_role_adaptive.html` through `v13_top_nav.html` in the original mockup history for the reasoning trail). Do not "clean up" or simplify away the patterns below without re-reading why they exist — each one is the direct fix for a specific failure mode found during design review.

## Interaction laws this template encodes

1. **One navigation system, one place.** Chat · To Do · Analytics · Approval · Email · New live as a single strip directly above the compose bar. Nothing else in the app duplicates navigation — no second tab row, no competing top-bar switcher. If a future product branch needs a new top-level destination, it gets added to this same strip; it does not get its own new nav pattern.
2. **Tap-first, AI-for-the-rest.** Any well-defined, common action (mark done, approve, nudge) is a direct tap — a checkbox or a one-tap button. The AI compose bar is reserved for genuinely open-ended requests (explain, draft, create, ask). Routing a simple "mark this done" through a chat message is the single worst ergonomic regression found during design review — never reintroduce it.
3. **Right-thumb ergonomics.** Primary repeated-action controls (checkboxes) sit on the right edge of each row, sized for thumb reach on a one-handed phone grip.
4. **Home Page has exactly three tabs, and they are universal, not rank-based.** To Do / Analytics / Approval are the same three tabs for every person in the company regardless of title — what's scoped is the *content* inside each tab (an individual contributor's Analytics tab shows personal pace; a manager's shows their team; a boss's shows the org), never the *navigation itself*. Do not reintroduce "Worker page / Manager page / Boss page" as separate screens — that was tried, and it silently re-encodes hierarchy into the UI in a way that reads as demeaning. Persona-based content scoping is correct; persona-gated navigation is not.
5. **Chat Page is a co-equal workspace, not an inbox bolted onto the side.** It carries its own real work surface: a pinned, always-visible VERIDIAN AI thread at the top (AI is one tap away regardless of filter), filter chips (All/Projects/Team/Boss) over one list rather than separate destinations, and — inside project threads — a pinned task card using the *exact same checkbox component* as Home Page's To Do tab, so finishing work from Chat feels identical to finishing it from Home.
6. **Instructions given in chat are tracked commitments, not just messages.** A message that assigns work gets a small instruction chip (assignee, due date, status). When the assignee's actual logged activity doesn't match what was asked, VERIDIAN surfaces the mismatch as a distinct AI-styled bubble (never confusable with a human message) to *the person who gave the instruction* — with a one-tap Nudge / It's fine resolution. The AI never auto-corrects or auto-escalates; it only makes the gap visible to the one person with standing to decide if it matters.
7. **Persona-preview (the "tap avatar" sheet) is a demo/QA device only.** In production there is no persona switcher — a user simply sees their own scoped content. Do not mistake this for a real feature when adapting the template for a live product.

## When building a new product branch on this template

- Reuse the CSS custom properties (`--navy`, `--saffron`, `--teal`, `--cream`, `--cloud`) as the base palette; branch-specific accent colors should be additive, not replacements, so the VERIDIAN AI family still reads as one platform.
- Reuse the bottom-zone structure (quick-strip + compose bar) verbatim. Swap only the *domain content* inside the To Do / Analytics / Approval panes and the Chat Page's filter set.
- If a branch genuinely needs a destination beyond Chat/To Do/Analytics/Approval/Email, it goes into the "+ New" sheet's option list or becomes a 7th quick-strip item — it does not spawn a second nav bar.
